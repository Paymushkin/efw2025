# SEO Оптимизация для Designer Runways

## Проблема
Данные Designer Runways загружаются динамически из Google Sheets, что не позволяет поисковым роботам их индексировать.

## Решение

### 1. Локальные данные для SSR
- Файл `constants/designerRunwaysData.ts` содержит локальные данные
- Эти данные используются при серверном рендеринге (SSR)
- Поисковые роботы видят эти данные в HTML и могут их индексировать

### 2. Обновление данных после загрузки
- После hydration (загрузки страницы в браузере) данные обновляются из Google Sheets
- Пользователи видят актуальные данные
- Локальные данные остаются в HTML для SEO

### 3. Автоматическая пересборка
- GitHub Actions workflow (`.github/workflows/daily-rebuild.yml`) запускается раз в сутки
- Скрипт `scripts/updateDesignerRunwaysData.ts` обновляет локальные данные из Google Sheets
- После обновления данных делается commit и push, что триггерит пересборку на Vercel

## Как это работает

1. **SSR (Server-Side Rendering)**:
   - При серверном рендеринге используются локальные данные из `constants/designerRunwaysData.ts`
   - Эти данные попадают в HTML, который видят поисковые роботы

2. **Hydration (Client-Side)**:
   - После загрузки страницы в браузере вызывается `fetchRunways()`
   - Данные обновляются из Google Sheets
   - Пользователи видят актуальные данные

3. **Автоматическое обновление**:
   - Каждый день в 00:00 UTC запускается GitHub Actions workflow
   - Скрипт загружает данные из Google Sheets и обновляет `constants/designerRunwaysData.ts`
   - Делается commit и push, что триггерит пересборку на Vercel

## Команды

- `npm run update-runways-data` - обновить локальные данные вручную
- `npm run build` - автоматически обновляет данные перед сборкой (через prebuild hook)

## Настройка

Для работы автоматической пересборки необходимо:
1. Убедиться, что GitHub Actions включены в репозитории
2. Убедиться, что Vercel настроен на автоматическую пересборку при push в main
